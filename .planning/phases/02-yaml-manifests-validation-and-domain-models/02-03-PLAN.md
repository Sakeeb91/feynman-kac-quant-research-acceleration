  <name>Task 1: Scenario generation from manifest including model sweep Cartesian product</name>
  <files>
    src/fk_quant_research_accel/orchestrator.py
    tests/test_orchestrator.py
  </files>
  <action>
    Add `generate_scenarios_from_manifest()` to `orchestrator.py` and extend the `Scenario` dataclass.

    **1. Extend `Scenario` dataclass:**
    Add an optional `model_config: dict[str, Any] | None = None` field to carry model/architecture sweep parameters per scenario. This is additive -- existing code that creates `Scenario(dim=5, ...)` without `model_config` still works.

    **2. Implement `generate_scenarios_from_manifest(manifest: ExperimentManifest) -> list[Scenario]`:**

    Generate the Cartesian product of:
    - `manifest.scenario_grid.dimensions`
    - `manifest.scenario_grid.volatilities`
    - `manifest.scenario_grid.correlations` (if scalars, each is a separate value; if matrix, single value)
    - `manifest.scenario_grid.option_types`
    - Model sweep axis: zip `manifest.model_sweep.architectures` with corresponding `hidden_sizes`, `activations`, `optimizers` (if provided). Each (architecture, hidden_size, activation, optimizer) tuple is one model config.

    For model sweep: if `hidden_sizes` is provided, it must have the same length as `architectures` (pair by index). Same for `activations` and `optimizers`. If `hidden_sizes` is None, omit from model_config dict. Build a list of `model_config` dicts from the zip.

    Use `itertools.product()` over all axes. For each combination, create a `Scenario` with the PDE params + model_config dict.

    Update `Scenario.as_parameters()` to include `model_config` in the returned dict when present.

    **3. Add tests to `tests/test_orchestrator.py`:**
    - `test_generate_scenarios_from_manifest_basic` -- 2 dims x 2 vols x 1 corr x 1 option = 4 scenarios
    - `test_generate_scenarios_from_manifest_with_model_sweep` -- 2 dims x 1 vol x 2 architectures = 4 scenarios, each with model_config
    - `test_generate_scenarios_from_manifest_model_config_in_params` -- scenario.as_parameters() includes model_config
    - `test_generate_scenarios_from_manifest_default_model_sweep` -- default ModelSweepConfig produces 1 model config entry (["default"])
    - `test_generate_scenarios_from_manifest_correlation_matrix` -- single correlation matrix produces 1 correlation value in product
    - `test_scenario_backward_compat` -- Scenario(dim=5, volatility=0.2, correlation=0.3) still works without model_config

    Ensure existing orchestrator tests still pass.
  </action>

  <done>
    Scenarios are generated from manifest via Cartesian product of PDE grid and model sweep axes. Model sweep is first-class in the Scenario dataclass. Existing scenario generation and orchestrator behavior preserved.

<task type="auto">
  <name>Task 2: CLI --manifest option with load, validate, hash, and execution wiring</name>
  <files>
    src/fk_quant_research_accel/cli.py
    tests/test_cli.py
  </files>
  <action>
    Add `--manifest` option to the `run-batch` CLI command and wire it through the full pipeline: load -> validate -> hash -> generate scenarios -> execute.

    **1. Update `run_batch_command` in `cli.py`:**

    Add `--manifest` parameter:
    ```python
    manifest: Path | None = typer.Option(
        None,
        "--manifest",
        exists=True,
        file_okay=True,
        dir_okay=False,
        readable=True,
        help="Path to experiment YAML manifest. Overrides all other config flags.",
    )
    ```

    **When `--manifest` is provided:**
    a. Import and call `load_manifest(str(manifest))` -- raises on invalid YAML/schema
    b. Import and call `content_hash(experiment)` -- log the hash
    c. Import and call `validate_manifest(experiment)` -- if errors, log each PreflightError (field, value, message) and `raise typer.Exit(code=1)`
    d. Call `generate_scenarios_from_manifest(experiment)` to build scenario list
    e. Build `BatchConfig` from `experiment.batch_config` fields
    f. Build `FKPinnClient` from `experiment.backend_url`
    g. Call `run_batch()` with scenarios, config, and manifest output paths (`experiment.output.artifacts_dir`, `experiment.output.db_path`)
    h. Pass `experiment.seed` through to `run_batch()`

    **When `--manifest` is NOT provided:**
    Existing CLI-flag-based path remains unchanged (backward compatible).

    **Make `--base-url` optional:** Currently `base_url` is required (`...`). Change to `base_url: str | None = typer.Option(None, "--base-url")`. When `--manifest` is not provided and `--base-url` is also not provided, raise `typer.BadParameter("--base-url is required when --manifest is not provided")`.

    **2. Add tests to `tests/test_cli.py`:**
    - `test_cli_manifest_option_exists_in_help` -- `run-batch --help` shows `--manifest`
    - `test_cli_manifest_loads_and_validates` -- create a valid YAML manifest in tmp, invoke CLI with --manifest, mock the client, verify scenario generation
    - `test_cli_manifest_preflight_fails_exits_1` -- create YAML with invalid params, invoke CLI, assert exit code 1
    - `test_cli_backward_compat_flags` -- existing flags still work without --manifest
    - `test_cli_base_url_required_without_manifest` -- --base-url omitted without --manifest raises error

    Use `typer.testing.CliRunner` for CLI tests. Mock `FKPinnClient` and `run_batch` to avoid real HTTP calls.

    **3. Log events:**
    - `manifest_loaded` with path and hash
    - `preflight_validation_failed` with error details (one event per error)
    - `preflight_passed` with scenario count
  </action>
