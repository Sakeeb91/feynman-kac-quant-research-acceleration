<objective>
Implement domain-specific pre-flight validation (CONF-05, CONF-06) that catches invalid parameter combinations before any simulation is submitted.

Purpose: Pre-flight validation prevents wasting compute on impossible scenarios. Researchers get clear error messages at manifest load time instead of discovering failures after hours of simulation. This is a prerequisite for concurrent execution in Phase 3 (invalid scenarios would waste concurrent slots).

Output: Pure-Python constraint validators (PSD check via Cholesky, range checks, dimension compatibility) and a preflight orchestrator that collects all errors from the manifest's scenario grid.
</objective>
