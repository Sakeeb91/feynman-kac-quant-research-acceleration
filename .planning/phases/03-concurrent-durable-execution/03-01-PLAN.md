   - Bump `CURRENT_SCHEMA_VERSION` from `1` to `2`.
   - In `init_db()`, change `sqlite3.connect(str(path))` to `sqlite3.connect(str(path), check_same_thread=False)`. This is required because the async orchestrator calls MetadataStore methods from AnyIO worker threads via `anyio.to_thread.run_sync()`. It is safe because: (a) WAL mode allows concurrent readers, (b) we have a single MetadataStore instance, (c) AnyIO's default thread limiter serializes access.
   - Add `1: _migrate_v1_to_v2` to the `migrations` dict in `_apply_migrations`.
   - Implement `_migrate_v1_to_v2(conn)`:
     ```python
     def _migrate_v1_to_v2(conn: sqlite3.Connection) -> None:
         """Add retry tracking and concurrency metadata for Phase 3."""
         conn.execute("ALTER TABLE scenario_runs ADD COLUMN retry_count INTEGER DEFAULT 0")
         conn.execute("ALTER TABLE scenario_runs ADD COLUMN max_retries INTEGER DEFAULT 3")
         conn.execute("ALTER TABLE batch_runs ADD COLUMN concurrency_limit INTEGER DEFAULT 1")
         conn.execute("ALTER TABLE batch_runs ADD COLUMN interrupted_at TEXT")
     ```

   - `update_batch_interrupted(self, batch_run_id: str, interrupted_at: str) -> None`: UPDATE batch_runs SET status = 'interrupted', interrupted_at = ? WHERE batch_run_id = ?. Commit after.
   - `update_scenario_retry_count(self, scenario_run_id: str, retry_count: int) -> None`: UPDATE scenario_runs SET retry_count = ? WHERE scenario_run_id = ?. Commit after.
   - Also update `create_batch_run()` to accept an optional `concurrency_limit: int = 1` parameter and INSERT it into the new column.

   - `is_retryable_error(exc: BaseException) -> bool`: Returns True for `httpx.TimeoutException`, `httpx.ConnectError`, `httpx.RemoteProtocolError`, and `httpx.HTTPStatusError` with `status_code >= 500`. Returns False for everything else (including 4xx errors).
   - `RETRY_DEFAULTS`: A dict with default retry config values: `{"max_attempts": 3, "initial_wait": 1.0, "max_wait": 60.0, "jitter": 5.0}`.
   - `make_retry_decorator(**overrides)`: Returns a tenacity `retry` decorator configured with `wait_exponential_jitter(initial=..., max=..., jitter=...)`, `stop_after_attempt(max_attempts)`, `retry=retry_if_exception(is_retryable_error)`, `reraise=True`. Merges RETRY_DEFAULTS with overrides.
   - Import structlog and log retries via a `before_sleep` callback that logs at warning level with attempt number and exception info.

2. **async_client.py**: Create async HTTP client mirroring `FKPinnClient` API.
   - `AsyncFKPinnClient` class with `__init__(self, base_url: str, timeout: float = 30.0, concurrency_limit: int = 20)`.
   - Internal `httpx.AsyncClient` with `httpx.Timeout(connect=10.0, read=timeout, write=10.0, pool=5.0)` and `httpx.Limits(max_connections=concurrency_limit + 5, max_keepalive_connections=concurrency_limit, keepalive_expiry=30.0)`.
   - Methods matching FKPinnClient: `async create_simulation(problem_id, parameters, training_config) -> dict`, `async get_simulation(simulation_id) -> dict`, `async get_result(simulation_id) -> dict`, `async list_problems() -> dict`.
   - `async aclose()` method and `__aenter__`/`__aexit__` for context manager support.
   - Do NOT add retry decorators to the client methods themselves -- retry is applied at the orchestrator level to allow per-operation retry configuration.
