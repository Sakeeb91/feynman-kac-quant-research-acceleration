Create `src/fk_quant_research_accel/async_orchestrator.py` implementing two public async functions: `run_batch_async` and `resume_batch_async`. Follow the patterns in 03-RESEARCH.md closely.

**`run_batch_async` function signature:**
```python
async def run_batch_async(
    client: AsyncFKPinnClient,
    scenarios: list[Scenario],
    batch_config: BatchConfig,
    poll_seconds: float = 2.0,
    max_wait_seconds: float = 1800.0,
    concurrency_limit: int = 20,
    max_retries: int = 3,
    artifacts_dir: str | Path = "artifacts",
    db_path: str | Path | None = None,
    seed: int | None = None,
    experiment_manifest_hash: str | None = None,
) -> list[dict[str, Any]]:
```

**Implementation details for `run_batch_async`:**

1. **Setup** (mirror sync `run_batch`):
   - Generate `batch_run_id`, create `ArtifactStore`, `MetadataStore`, batch directory, `RunManifest`.
   - Create batch_run in SQLite with `concurrency_limit` parameter.
   - Register all scenarios in SQLite (create_scenario_run for each).
   - Use `async with client:` context manager for proper cleanup.

2. **Concurrent execution with AnyIO:**
   - Create `CapacityLimiter(concurrency_limit)`.
   - Use `async with create_task_group() as tg:` and `tg.start_soon(_execute_scenario_safe, ...)` for each scenario.
   - Each scenario runs independently within the limiter's bounded concurrency.

3. **`_execute_scenario_safe` (internal):** This is the critical fault-isolation wrapper.
   - Acquires the CapacityLimiter slot via `async with limiter:`.
   - Calls `_submit_and_poll_scenario(...)` inside a `try/except Exception`.
   - On success: persists result to SQLite and artifact store.
   - On failure: builds failure record via `_build_failure_record`, persists to SQLite and artifact store, logs error. Does NOT re-raise -- this prevents task group cancellation of siblings.
   - Appends the record to a shared `results: list` (thread-safe because AnyIO tasks are cooperative, not preemptive).

4. **`_submit_and_poll_scenario` (internal):** Handles one scenario end-to-end.
   - **Submit:** Call `client.create_simulation(...)`. Wrap with retry using `make_retry_decorator(max_attempts=max_retries)`. Update scenario status to 'submitted' in SQLite via `await anyio.to_thread.run_sync(partial(store.update_scenario_status, ...))`.
   - **Poll:** Loop with `await anyio.sleep(poll_seconds)` until terminal status or timeout. Each `client.get_simulation(...)` call is wrapped with retry. Add small jitter to poll interval: `poll_seconds + random.uniform(0, poll_seconds * 0.3)` to avoid synchronized polling bursts.
   - **Collect result:** Call `client.get_result(...)` (with retry). Extract metrics, compute score via `compute_score()`, build result record matching the sync orchestrator's record format.
   - **Checkpoint fetch:** Attempt checkpoint retrieval (mirror `_fetch_checkpoint` from sync orchestrator but using httpx for the checkpoint URL download). Non-fatal on failure.
   - **Persist:** Write result to SQLite and `result.json` via artifact store -- both wrapped in `to_thread.run_sync`.

5. **All SQLite calls** must go through `await anyio.to_thread.run_sync(partial(store.method, ...))`. Use `functools.partial` for calls with arguments.

6. **After task group completes:**
   - Update batch status to 'completed' via SQLite.
   - Log summary (total, completed, failed).
   - Close MetadataStore.
   - Return sorted results (by score ascending, matching sync orchestrator).

**`resume_batch_async` function signature:**
```python
async def resume_batch_async(
    client: AsyncFKPinnClient,
    batch_run_id: str,
    concurrency_limit: int = 20,
    max_retries: int = 3,
    poll_seconds: float = 2.0,
    max_wait_seconds: float = 1800.0,
    force: bool = False,
    db_path: str | Path | None = None,
    artifacts_dir: str | Path = "artifacts",
) -> list[dict[str, Any]]:
```

**Implementation details for `resume_batch_async`:**
1. Open MetadataStore, verify batch_run_id exists (raise ValueError if not).
2. If `force`: get ALL scenario_runs. Otherwise: call `get_incomplete_scenario_runs(batch_run_id)`.
3. If no incomplete scenarios: log info and return empty list.
4. Reconstruct `Scenario` objects from `scenario_json` column (JSON -> dict -> Scenario(**params)).
5. Reconstruct `BatchConfig` from batch_run's `config_json`.
6. Re-execute incomplete scenarios using the same `CapacityLimiter` + `create_task_group` pattern as `run_batch_async` (extract shared execution into a helper `_execute_scenarios_concurrent`).
7. Update batch status on completion, return sorted results.

**Imports:** Use `from __future__ import annotations`. Import from `async_client`, `retry`, `store`, `models`, `reporting` as needed. Import `anyio`, `structlog`, `json`, `random`, `functools.partial`.

**CRITICAL:** Each scenario in the task group MUST catch `Exception` (not `BaseException`) to prevent one failure from cancelling all siblings. Let `KeyboardInterrupt` and `SystemExit` propagate naturally (they are `BaseException` subclasses).

2. **test_concurrent_execution_respects_limit**: 5 scenarios, concurrency_limit=2. Mock client adds a small `anyio.sleep(0.01)` in create_simulation. Track concurrent execution count via a shared counter incremented on entry and decremented on exit. Assert max concurrent never exceeds 2.

3. **test_single_failure_does_not_cancel_siblings**: 3 scenarios. Mock client raises `RuntimeError("bad scenario")` for scenario index 1. Verify: 3 results returned (2 completed, 1 failed). 4. **test_transient_error_retried**: 1 scenario. Mock client's `get_simulation` raises `httpx.TimeoutException` on first call, then returns completed. Verify: scenario completes successfully (retry handled the transient error).

8. **test_resume_batch_async_nonexistent_batch**: Call with a nonexistent batch_run_id. Verify ValueError is raised.
