  <files>src/fk_quant_research_accel/cli.py</files>
  <action>
Modify `src/fk_quant_research_accel/cli.py` to:

1. **Add imports:** Import `anyio`, `functools.partial`, `AsyncFKPinnClient` from `async_client`, and `run_batch_async`, `resume_batch_async` from `async_orchestrator`.

2. **Update `run-batch` command:** Add two new Typer options:
   - `--concurrency: int = typer.Option(20, "--concurrency", min=1, max=100, help="Max concurrent scenario executions")`
   - `--max-retries: int = typer.Option(3, "--max-retries", min=0, max=10, help="Max retry attempts per transient HTTP error")`

   Replace the call to sync `run_batch(...)` with async execution:
   ```python
   rows = anyio.run(
       partial(
           run_batch_async,
           client=AsyncFKPinnClient(base_url=client.base_url, concurrency_limit=concurrency),
           scenarios=scenarios,
           batch_config=config,
           poll_seconds=effective_poll_seconds,
           max_wait_seconds=effective_max_wait_seconds,
           concurrency_limit=concurrency,
           max_retries=max_retries,
           artifacts_dir=artifacts_dir,
           db_path=db_path,
           seed=seed,
           experiment_manifest_hash=experiment_manifest_hash,
       )
   )
   ```

   Note: The `AsyncFKPinnClient` is instantiated here with `base_url` from either the manifest or `--base-url` flag. The `run_batch_async` function handles the `async with client:` lifecycle internally.

   Keep the sync `FKPinnClient` import for now but it is no longer used in `run-batch`. Remove the `from .client import FKPinnClient` import if it becomes unused, OR keep it if other code paths (tests, etc.) still reference it. If manifest mode constructs FKPinnClient, change it to just extract `base_url` and pass it to AsyncFKPinnClient instead.

3. **Add `resume-batch` command:**
   ```python
   @app.command("resume-batch")
   def resume_batch_command(
       batch_run_id: str = typer.Argument(
           ..., help="Batch run ID to resume (from previous run-batch output)"
       ),
       force: bool = typer.Option(
           False, "--force", help="Re-run ALL scenarios, including completed ones"
       ),
       concurrency: int = typer.Option(
           20, "--concurrency", min=1, max=100,
           help="Max concurrent scenario executions"
       ),
       max_retries: int = typer.Option(
           3, "--max-retries", min=0, max=10,
           help="Max retry attempts per transient HTTP error"
       ),
       base_url: str = typer.Option(
           ..., "--base-url", help="FK PINN backend URL"
       ),
       poll_seconds: float = typer.Option(2.0, "--poll-seconds"),
       max_wait_seconds: float = typer.Option(1800.0, "--max-wait-seconds"),
       db_path: str = typer.Option("artifacts/experiments.db", "--db-path"),
       artifacts_dir: str = typer.Option("artifacts", "--artifacts-dir"),
       output: str = typer.Option("artifacts/resume_results.csv", "--output"),
   ) -> None:
   ```

   Implementation:
   - Log the resume attempt with batch_run_id and force flag.
   - Create AsyncFKPinnClient with base_url and concurrency_limit.
   - Call `anyio.run(partial(resume_batch_async, client=..., batch_run_id=..., ...))`.
   - Catch `ValueError` (nonexistent batch) and exit with code 1 + error log.
   - Write results to CSV via `write_csv(rows, output)`.
   - Log top results and completion summary.

4. **Preserve backward compatibility:** The `run-batch` command must still accept all existing flags (--base-url, --manifest, --dimensions, --volatilities, etc.). The new `--concurrency` and `--max-retries` flags are additive. Legacy flag-based runs (no --manifest) should also use the async path now.

5. **Remove unused sync imports** if `run_batch` (sync) is no longer called from CLI. Keep the sync `orchestrator.py` module itself intact -- it may be used by tests or other consumers.
  </action>
