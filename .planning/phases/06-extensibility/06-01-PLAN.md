Create `src/fk_quant_research_accel/problems/protocol.py`:
- `ProblemParams(BaseModel)` -- base class for problem parameter schemas
- `ProblemSpec` -- `@runtime_checkable` Protocol with:
  - `problem_id: str` (property)
  - `param_schema: type[ProblemParams]` (property)
  - `generate_scenarios(grid_config: dict[str, Any], model_configs: list[dict[str, Any]]) -> list[dict[str, Any]]`
  - `validate(params: dict[str, Any]) -> list[str]`
  - `default_scorer(record: dict[str, Any]) -> float`
  - `default_pareto_objectives() -> list[str]`
  - `supports_scoring_strategy(strategy: str) -> bool`
- `BaseProblemSpec` -- concrete class with default implementations:
  - `validate`: uses `self.param_schema.model_validate(params)`, returns `[]` on success, `[str(exc)]` on failure
  - `default_scorer`: returns `float(train_loss)` for completed records, `float("inf")` otherwise
  - `default_pareto_objectives`: returns `["train_loss", "grad_norm"]`
  - `supports_scoring_strategy`: returns `True`

Create `src/fk_quant_research_accel/problems/registry.py`:
- `_PROBLEM_REGISTRY: dict[str, ProblemSpec] = {}`
- `register_problem(spec: ProblemSpec) -> ProblemSpec` -- registers spec, raises `ValueError` on duplicate
- `get_problem_spec(problem_id: str) -> ProblemSpec` -- lazy-imports `black_scholes` and `harmonic_oscillator` modules to trigger registration, then looks up; raises `ValueError` with valid IDs and `difflib.get_close_matches` suggestion on miss
- `list_problem_ids() -> list[str]` -- triggers lazy registration, returns sorted keys
- Mirror the pattern from `scoring/registry.py` (lazy import to trigger decorators/registrations)

Create `src/fk_quant_research_accel/problems/__init__.py`:
- Re-export: `ProblemSpec`, `BaseProblemSpec`, `ProblemParams`, `get_problem_spec`, `register_problem`, `list_problem_ids`
