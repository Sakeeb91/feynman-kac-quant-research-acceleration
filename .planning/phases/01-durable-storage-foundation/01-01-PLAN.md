    1. Create `src/fk_quant_research_accel/store/__init__.py` exporting MetadataStore, ArtifactStore, init_db.

    2. Create `src/fk_quant_research_accel/store/migrations.py`:
       - `CURRENT_SCHEMA_VERSION = 1`
       - `init_db(db_path: str | Path) -> sqlite3.Connection`: connect, enable WAL mode (`PRAGMA journal_mode=WAL`), enable foreign keys (`PRAGMA foreign_keys=ON`), set busy timeout (`PRAGMA busy_timeout=5000`), set synchronous=NORMAL (`PRAGMA synchronous=NORMAL`), set `row_factory = sqlite3.Row`, read current `PRAGMA user_version`, call `_apply_migrations()`, return connection.
       - `_apply_migrations(conn, current_version)`: iterate from current_version to CURRENT_SCHEMA_VERSION, call each migration function, set `PRAGMA user_version = v+1`, commit after each.
       - `_migrate_v0_to_v1(conn)`: create `batch_runs` table (batch_run_id TEXT PRIMARY KEY, created_at TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'running', config_json TEXT NOT NULL, manifest_schema_version INTEGER NOT NULL DEFAULT 1, git_sha TEXT, git_dirty INTEGER, python_version TEXT, os_info TEXT, seed INTEGER, scenario_count INTEGER NOT NULL, completed_count INTEGER NOT NULL DEFAULT 0, failed_count INTEGER NOT NULL DEFAULT 0, artifact_path TEXT NOT NULL) and `scenario_runs` table (scenario_run_id TEXT PRIMARY KEY, batch_run_id TEXT NOT NULL REFERENCES batch_runs(batch_run_id), scenario_json TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'pending', simulation_id TEXT, result_json TEXT, score REAL, error_message TEXT, created_at TEXT NOT NULL, started_at TEXT, completed_at TEXT, checkpoint_path TEXT). Create indexes: idx_scenario_batch on scenario_runs(batch_run_id), idx_scenario_status on scenario_runs(status). Use `CREATE TABLE IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS` for idempotency (research Pitfall 5).

    3. Create `src/fk_quant_research_accel/store/metadata.py`:
       - `MetadataStore` class with `__init__(self, db_path: str | Path)` that calls `init_db()`.
       - `create_batch_run(self, batch_run_id, created_at, config_json, manifest_schema_version, git_sha, git_dirty, python_version, os_info, seed, scenario_count, artifact_path) -> None`: INSERT into batch_runs, commit.
       - `create_scenario_run(self, scenario_run_id, batch_run_id, scenario_json, created_at) -> None`: INSERT into scenario_runs with status='pending', commit.
       - `update_scenario_status(self, scenario_run_id, status, simulation_id=None, started_at=None) -> None`: UPDATE scenario_runs SET status, simulation_id, started_at WHERE scenario_run_id. Commit.
       - `persist_scenario_result(self, scenario_run_id, status, result_json, score=None, error_message=None, completed_at=None, checkpoint_path=None) -> None`: UPDATE scenario_runs SET status, result_json, score, error_message, completed_at, checkpoint_path WHERE scenario_run_id. Also UPDATE batch_runs: increment completed_count or failed_count based on status. Commit explicitly after EACH call (research Pitfall 2 -- crash-safety requires commit per scenario).
       - `get_batch_run(self, batch_run_id) -> dict | None`: SELECT * FROM batch_runs WHERE batch_run_id, return dict or None.
       - `get_scenario_runs(self, batch_run_id) -> list[dict]`: SELECT * FROM scenario_runs WHERE batch_run_id, return list of dicts.
       - `update_batch_status(self, batch_run_id, status) -> None`: UPDATE batch_runs SET status WHERE batch_run_id. Commit.
       - `close(self) -> None`: close the connection.
       - Keep `check_same_thread=True` (default) -- Phase 1 is single-threaded (research Pitfall 1).

    4. Create `src/fk_quant_research_accel/store/artifacts.py`:
       - `ArtifactStore` class with `__init__(self, root_dir: str | Path)` that sets `self.root = Path(root_dir)`.
       - `create_batch_dir(self, batch_run_id: str) -> Path`: create and return `self.root / batch_run_id` directory.
       - `create_scenario_dir(self, batch_run_id: str, scenario_run_id: str) -> Path`: create and return `self.root / batch_run_id / scenario_run_id`.
       - `get_scenario_dir(self, batch_run_id: str, scenario_run_id: str) -> Path`: return path (no create).
       - `atomic_write_json(self, path: Path, data: dict) -> None`: write to .tmp then rename (research Pattern: atomic file write).
       - `atomic_write_bytes(self, path: Path, data: bytes) -> None`: write bytes to .tmp then rename (for checkpoint files).
       - `atomic_write_text(self, path: Path, content: str) -> None`: write text to .tmp then rename.

       - Use `tmp_path` pytest fixture for all filesystem tests.
