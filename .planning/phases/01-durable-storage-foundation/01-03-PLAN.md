  <action>
    Modify `src/fk_quant_research_accel/orchestrator.py` to wire in durable storage:

    1. **Keep existing public API intact.** `Scenario`, `BatchConfig`, `generate_black_scholes_scenarios` remain unchanged. `run_batch()` gets new optional parameters but existing callers still work.

    2. **Add imports:** structlog, json, datetime, uuid. Import from models (BatchRunId, ScenarioRunId, generate_batch_run_id, generate_scenario_run_id, RunManifest, ManifestMetadata, ReproducibilityInfo, ScenarioStatus). Import from store (MetadataStore, ArtifactStore). Import from models.manifest (write_manifest, capture_git_info, capture_environment).

    3. **Add optional storage parameters to `run_batch()`:**
       ```python
       def run_batch(
           client: FKPinnClient,
           scenarios: list[Scenario],
           batch_config: BatchConfig,
           poll_seconds: float = 1.5,
           max_wait_seconds: float = 1800.0,
           # New optional parameters:
           artifacts_dir: str | Path = "artifacts",
           db_path: str | Path | None = None,  # defaults to artifacts_dir/experiments.db
           seed: int | None = None,
       ) -> list[dict[str, Any]]:
       ```
       Making these optional preserves backward compatibility -- existing callers don't need to pass them.

    4. **At batch start (before any scenario is submitted):**
       - Generate `batch_run_id = generate_batch_run_id()`
       - Bind structlog context: `log = structlog.get_logger().bind(batch_run_id=batch_run_id)`
       - Create `ArtifactStore(artifacts_dir)` and `artifact_store.create_batch_dir(batch_run_id)`
       - If `db_path` is None, set to `Path(artifacts_dir) / "experiments.db"`
       - Create `MetadataStore(db_path)`
       - Capture reproducibility: `git_sha, git_dirty = capture_git_info()`, `env = capture_environment()`
       - Build `ReproducibilityInfo(git_sha=git_sha, git_dirty=git_dirty, python_version=env["python_version"], os_info=env["os_info"], seed=seed, packages=env["packages"])`
       - Build `RunManifest(batch_run_id=batch_run_id, created_at=datetime.now(timezone.utc), reproducibility=repro_info, batch_config=batch_config.to_payload(), scenarios=[s.as_parameters() for s in scenarios], backend_url=client.base_url)`
       - Call `write_manifest(manifest, artifact_store.root)` -- atomic write of manifest.yaml
       - Call `metadata_store.create_batch_run(...)` with all fields from the manifest
       - Log: `log.info("batch_started", scenario_count=len(scenarios), artifact_dir=str(artifact_store.root / batch_run_id))`

    5. **Generate scenario_run_ids upfront and register them in the database:**
       - For each scenario, generate `scenario_run_id = generate_scenario_run_id()`.
       - Call `metadata_store.create_scenario_run(scenario_run_id, batch_run_id, json.dumps(scenario.as_parameters()), datetime.now(timezone.utc).isoformat())`.
       - Create the scenario artifact directory: `artifact_store.create_scenario_dir(batch_run_id, scenario_run_id)`.
       - Store the mapping: `scenario_map: list[tuple[Scenario, ScenarioRunId]] = [...]`.

    6. **Submit all scenarios** (same as current behavior but with logging):
       - For each (scenario, scenario_run_id) in scenario_map:
         - `simulation = client.create_simulation(...)` (existing code)
         - `metadata_store.update_scenario_status(scenario_run_id, "submitted", simulation_id=simulation["id"], started_at=datetime.now(timezone.utc).isoformat())`
         - `submitted.append((scenario, scenario_run_id, simulation["id"]))`
         - `log.info("scenario_submitted", scenario_run_id=scenario_run_id, simulation_id=simulation["id"])`

    7. **Poll and collect results with incremental writes** (replace the current accumulate-in-memory pattern):
       - For each (scenario, scenario_run_id, simulation_id) in submitted:
         - Wrap in try/except to catch ALL exceptions (TimeoutError, requests errors, etc.)
         - On success:
           - Extract metrics from result (same as current: train_loss, val_loss, grad_norm, lr, progress)
           - Compute score using `compute_score(record)`
           - Build the record dict (same fields as current)
           - **Persist to SQLite immediately:** `metadata_store.persist_scenario_result(scenario_run_id, "completed", json.dumps(record), score=record["score"], completed_at=datetime.now(timezone.utc).isoformat())`
           - **Persist to filesystem immediately:** `artifact_store.atomic_write_json(scenario_dir / "result.json", record)`
           - **Attempt checkpoint fetch:** call `_fetch_checkpoint(client, simulation_id, scenario_dir)` in a try/except (see step 8). If checkpoint retrieved, update `metadata_store.persist_scenario_result()` again with checkpoint_path. If not, leave it as None.
           - `log.info("scenario_completed", scenario_run_id=scenario_run_id, score=record["score"])`
         - On failure (any exception):
           - `error_msg = str(e)`
           - Build a failure record with status="failed", score=inf, error_message=error_msg
           - **Persist failure to SQLite:** `metadata_store.persist_scenario_result(scenario_run_id, "failed", json.dumps(failure_record), error_message=error_msg, completed_at=datetime.now(timezone.utc).isoformat())`
           - **Persist failure to filesystem:** `artifact_store.atomic_write_json(scenario_dir / "result.json", failure_record)`
           - `log.error("scenario_failed", scenario_run_id=scenario_run_id, error=error_msg, exc_info=True)`
           - **Do NOT re-raise** -- continue to next scenario. Failed scenarios are recorded, not dropped (EXEC-05).
         - Append record to `records` list for the return value.

    8. **Add private `_fetch_checkpoint()` function:**
       ```python
       def _fetch_checkpoint(
           client: FKPinnClient,
           simulation_id: str,
           scenario_dir: Path,
       ) -> Path | None:
       ```
       - Create `scenario_dir / "checkpoint"` directory.
       - Try to call `client.get_result(simulation_id)` and check if the response contains a checkpoint URL or data. Since the FK backend checkpoint API is undocumented (research Open Question 1), implement a best-effort approach:
         - Check if `result["item"]` has a `"checkpoint_url"` key. If so, download it.
         - Check if `result["item"]` has a `"checkpoint"` key with base64 data. If so, decode and write.
         - If neither exists, log a debug message and return None.
       - On any exception: log warning `"checkpoint_fetch_failed"`, return None.
       - On success: atomic_write_bytes to `checkpoint/model_checkpoint.pt`, return path.
       - **This function MUST NOT raise** -- checkpoint failure should never fail a scenario (research Pitfall 6).

    9. **At batch end:**
       - Update batch status: `metadata_store.update_batch_status(batch_run_id, "completed")`
       - `log.info("batch_completed", total=len(records), completed=sum(1 for r in records if r["status"]=="completed"), failed=sum(1 for r in records if r["status"]!="completed"))`
       - Close metadata store: `metadata_store.close()`
       - Return `sorted(records, key=lambda row: row["score"])` (same as current).

    10. Update `src/fk_quant_research_accel/__init__.py`:
        - Keep existing exports. The run_batch signature has changed (new optional params) but is backward compatible.
