# Testing Patterns

**Analysis Date:** 2026-02-19

## Test Framework

**Runner:**
- pytest 8.0.0+ (configured in `pyproject.toml`)
- Config file: `pyproject.toml` with `[tool.pytest.ini_options]`
- Python path: src/ (allows imports directly from modules)

**Assertion Library:**
- pytest built-in assertions (native `assert` statements)

**Run Commands:**
```bash
pytest -q              # Run all tests, quiet mode
pytest tests/          # Run specific test directory
pytest tests/test_reporting.py::test_name  # Run specific test
pytest --co           # Collect tests without running (dry-run)
```

## Test File Organization

**Location:**
- Co-located with source: test files in `/tests/` directory parallel to `/src/`
- Convention: `test_*.py` file names matching module names
- Test directory structure mirrors source structure when necessary

**Naming:**
- Test modules: `test_*.py` format (e.g., `test_reporting.py`)
- Test functions: `test_*()` format (e.g., `test_compute_score_completed_has_finite_value()`)
- Descriptive names indicating what is tested and expected outcome

**Structure:**
```
tests/
├── test_reporting.py      # Tests for reporting.py module
└── __pycache__/           # Generated by pytest
```

Currently only 3 tests in the codebase; growth expected as functionality expands.

## Test Structure

**Suite Organization:**
```python
# tests/test_reporting.py - example of current pattern

def test_generate_black_scholes_scenarios_cross_product() -> None:
    scenarios = generate_black_scholes_scenarios(
        dimensions=[5, 10],
        volatilities=[0.15, 0.2],
        correlations=[0.0, 0.3],
        option_types=["call", "put"],
    )
    assert len(scenarios) == 16
```

**Patterns:**
- No test classes (using module-level functions)
- No setup/teardown fixtures (none needed yet; state is immutable)
- Each test is standalone and independent
- Type hints included on test functions: `-> None`

## Mocking

**Framework:** None configured

**Status:**
- Current tests do not require mocking
- External dependencies (HTTP client, file I/O) are not mocked in existing tests
- Mock framework (pytest-mock, unittest.mock) available via pytest but not currently used

**What to Mock (Future Pattern):**
- HTTP calls in `FKPinnClient` should be mocked using `unittest.mock.patch` or `pytest-mock`
- File I/O in `write_csv()` could be mocked if testing CSV format without filesystem access
- Time-based polling in `wait_until_terminal()` should be mocked to avoid slow tests

**What NOT to Mock:**
- Pure functions like `compute_score()` - test directly without mocking
- Data classes like `Scenario` and `BatchConfig` - no mocking needed, they're transparent

## Test Types

**Unit Tests:**
- Scope: Individual functions in isolation
- Approach: Test pure functions with known inputs and outputs
- Examples: `test_compute_score_*()` tests
- Coverage: reporting.py functions, orchestrator utility functions

**Integration Tests:**
- Scope: Multiple components together (would include HTTP client + backend)
- Approach: Test full workflow (client → server → result parsing)
- Status: Not currently implemented
- Recommended: Add when backend integration is stable

**E2E Tests:**
- Framework: Not used
- Status: Could use pytest with live backend for full workflow testing
- Example scenario: Submit batch, wait for completion, verify CSV output

## Common Patterns

**Assertion Style:**
```python
# Simple equality
assert len(scenarios) == 16

# Range checks
assert score > 0.0
assert score < 1.0

# Equality with special values
assert score == float("inf")
```

**Async Testing:**
- Not applicable (no async code in current codebase)
- Future: If polling moves to async, use `pytest-asyncio`

**Error Testing (Current Gaps):**
- No tests for exception cases
- Future pattern (when needed):

**Mocking Pattern (Future):**
```python
from unittest.mock import patch, MagicMock

def test_create_simulation_calls_correct_endpoint() -> None:
    with patch("fk_quant_research_accel.client.requests.post") as mock_post:
        mock_post.return_value.json.return_value = {"id": "sim_123"}

        result = client.create_simulation("black_scholes", {}, {})

        assert result["id"] == "sim_123"
        mock_post.assert_called_once()
```

## CI/CD Integration

**CI Pipeline:** `.github/workflows/ci.yml`

**Test Execution:**
1. Runs on Python 3.11 (ubuntu-latest)
2. Installs dev dependencies: `pip install -e ".[dev]"`
3. Runs linting: `ruff check src tests`
4. Runs tests: `pytest -q`
5. Tests must pass before PR merge

**Exit Codes:**
- 0: All tests pass
- Non-zero: Any test or lint failure blocks merge
